### 3. DeFi vs Traditional Finance / DeFi vs 传统金融（High Frequency for Essay / 短答高频）

| DeFi / 去中心化金融 | Traditional Finance / 传统金融 |

### 9. PBM vs Programmable Money / 目的绑定货币 vs 可编程货币

- **PBM (Purpose Bound Money)** = uses a **wrapper** to add usage conditions to money / 用外层包装器给钱加使用条件
- Programmable Money = programs logic **directly into the token** / 把逻辑直接写进代币
- **Key difference / 关键区别**：PBM does not modify the money token itself / PBM 不改 money token 本身
- PBM is associated with **Project Orchid / SGD Testnet**

### 10. Bank Deposit Tokens vs Stablecoins / 银行存款代币 vs 稳定币

- **Key difference / 关键区别**：Bank deposit tokens are subject to **current banking regulations** / 银行存款代币受现有银行监管约束
- Both are issued on blockchain, have backing mechanisms, represent fiat value / 都发行在区块链上、有价值支持机制、代表法币价值
- Bank deposit tokens = liabilities of **regulated banks** / 银行存款代币 = 受监管银行的负债

### 2. RWA Tokenization Process — Must-Memorize Order / RWA 代币化流程（必背顺序）

**Legal → Regulatory → Structural → Technical → Operational**

1. **Asset & Rights Structuring** — Define what is being tokenized (ownership, income right, redemption right) / 定义被代币化的资产和法律权利
2. **Regulatory Classification & Jurisdictional Analysis** — Determine applicable laws and licensing / 监管分类和司法辖区分析
3. **Legal Vehicle Formation** — Establish SPV, fund, or trust / 设立 SPV/基金/信托等法律载体
4. **Offering Documentation & Disclosure** — Prepare prospectus, risk disclosures / 发行文件和信息披露
5. **Token Design & Smart Contract Architecture** — Encode legal rights into programmable logic / 代币设计和智能合约架构
6. **Issuance, Distribution & Onboarding** — Issue tokens to compliant investors / 发行、分发和合规准入
7. **Post-issuance Lifecycle Management** — Audit, income distribution, disclosures / 生命周期管理（审计、付息、赎回）
8. **Secondary Trading, Settlement & Redemption** — Enable liquidity while maintaining compliance / 二级交易、结算和赎回

> **Core Trap / 核心陷阱**：RWA tokenization is **legal and regulatory first, not technology first** / RWA 代币化先法律/监管，后技术。Past Year: "What is the best first step?" → **Define the asset and legal rights being tokenized**

### 3. Gold-backed Token Architecture / 黄金代币化架构

**What blockchain solves / 区块链解决的是**：
- **Ledger integrity** (single source of truth for token supply) / 账本完整性（代币供应的单一事实来源）
- **Supply verification** (immutable issuance/redemption records) / 供应验证（不可篡改的发行/赎回记录）
- **Coordination** (atomic settlement, no mint without backing) / 协调（原子结算，无支持不可铸造）

**What blockchain does NOT solve / 区块链不解决的**：
- Physical gold storage / 实物黄金存储

**On-chain components / 链上组件**：
- GoldToken contract (balances, transfer restrictions, mint/burn) / 代币合约（余额、转让限制、铸造/销毁）
- Issuance & Redemption Controller (validates mint/burn, enforces 1:1 backing) / 发行与赎回控制器
- Governance / Emergency Control Contract (pause, upgrade) / 治理/紧急控制合约

**Off-chain components / 链下组件**：
- **Gold vault management system** / 金库管理系统
- **Auditor** / 审计员
- **Reserve oracle** / 储备预言机
- **Issuer operations** / 发行方运营

### 6. DAI / MakerDAO Mechanics / DAI / MakerDAO 机制

- **Over-collateralized** crypto assets locked in smart contracts / 超额加密抵押，锁定在智能合约中
- Requires **price oracle** to track collateral value / 需要预言机追踪抵押品价格
- If collateralization falls below required threshold → **liquidation** / 抵押率低于阈值 → 清算
- Benefits / 好处：Transparent, non-custodial, DeFi composable / 透明、非托管、DeFi 可组合
- Trade-offs / 代价：**Capital inefficient** (over-collateralization required) / 资本效率低、**Oracle dependency** / 预言机依赖、**Liquidation risk** / 清算风险

### 7. Terra / UST Death Spiral — Must-Know / Terra / UST 死亡螺旋（必考重点）

**Dual-token system / 双代币系统**：
- **UST** = stable unit / 稳定单位
- **LUNA** = volatile absorber + staking token / 波动吸收器 + 质押代币
- No collateral pool guaranteeing redemption / 没有抵押池保证赎回

**Contraction mechanism / 收缩机制**：
When UST < $1 (UST 低于 $1 时)：
1. Arbitrageur buys cheap UST / 套利者买入便宜 UST
2. Burns 1 UST / 销毁 1 UST
3. Mints $1 worth of LUNA / 铸造价值 $1 的 LUNA
4. Creates **LUNA sell pressure** / 制造 LUNA 卖压

**Death spiral sequence / 死亡螺旋链条**：
1. Large UST exits, especially from Anchor / 大量 UST 退出（尤其从 Anchor）
2. UST burned → massive LUNA minting / 销毁 UST → 大量铸造 LUNA
3. LUNA supply explodes → price collapses / LUNA 供应爆炸 → 价格暴跌
4. Lower LUNA price → even more LUNA must be minted per UST redemption / 越跌越需要铸造更多 LUNA
5. Confidence collapses → peg becomes unenforceable / 信心崩溃 → 锚定无法维持

**Anchor Protocol**：~20% APY created major demand sink for UST, but yield was **unsustainable** / ~20% 年化收益创造 UST 需求重心，但收益不可持续

**LFG (Luna Foundation Guard) Weakness**：BTC reserves were **discretionary support**, NOT protocol-enforced hard collateral / BTC 储备是自由裁量支持，不是协议强制硬抵押

**Redesign ideas / 重设计思路**：
- **Circuit breakers** / 熔断机制
- **Capped LUNA issuance** / LUNA 发行上限
- **Dynamic redemption haircuts** / 动态赎回折扣
- **Redemption queue** / 赎回队列
- **Rule-based exogenous reserves** / 规则化外生储备

### 8. Stablecoin Trilemma / 稳定币三难困境

**Stability vs Independence vs Costs / 稳定性 vs 独立性 vs 成本**

- Strong exogenous collateral → more stable but higher cost / 强外生抵押品 → 更稳定但成本更高
- More independent (no fiat/banks) → often needs over-collateralization / 更独立 → 通常需要超额抵押
- Low cost → may become reflexive and unstable under stress / 低成本 → 压力下可能反身性崩溃

### 1. DeFi Composability = Financial Lego / DeFi 可组合性 = 金融乐高

### 4. AMM / CFMM Core Formula / AMM/CFMM 核心公式

**x × y = k** (Constant Product Invariant / 常数乘积不变量)
- x = reserve of token A / 代币 A 的储备
- y = reserve of token B / 代币 B 的储备
- k = constant / 常数

Key points / 核心要点：
- Price comes from **reserve ratio**, **no oracle needed** / 价格来自储备比例，不需要预言机
- **Arbitrage** aligns pool price with market price / 套利让池子价格接近市场价格
- LPs earn **trading fees** / LP 赚取交易费

**Slippage / 滑点**：
- Slippage = difference between expected and actual execution price / 预期成交价与实际成交价的差异
- Increases with **trade size relative to pool depth** / 交易越大（相对池子深度），滑点越高
- Slippage is **structural**, not accidental / 滑点是结构性的，不是偶发错误

**Numerical example / 计算例题**：
ETH/USDC pool: 100 ETH + 200,000 USDC → Spot price = **2,000 USDC/ETH**, k = 20,000,000
- Swap 1 ETH: receive ~1,980 USDC (slippage ~1%) / 换 1 ETH 得约 1,980 USDC
- Swap 30 ETH: receive ~46,154 USDC (expected 60,000; slippage ~23%) / 换 30 ETH 得约 46,154 USDC

### 5. CDP vs Pooled Lending / CDP 抵押债务头寸 vs 池化借贷

| CDP (e.g., MakerDAO) | Pooled Lending (e.g., Compound, Aave) |
|---|---|
| Lock collateral → **mint new debt tokens** / 锁抵押品 → 铸造新债务代币 | **Borrow existing assets** from a pool / 从资金池借已有资产 |
| No direct lender needed / 不需要直接贷款人 | Interest rate varies with **utilization rate** / 利率随利用率变化 |

### 6. cToken Interest Accrual — Key Trap / cToken 计息机制（关键陷阱）

- User supplies assets to Compound → receives cTokens / 用户向 Compound 供应资产 → 获得 cToken
- cToken is an **ERC-20 token** representing proportional claim on underlying pool / cToken 是 ERC-20 代币，代表对底层资金池的按比例索取权
- **Number of cTokens stays CONSTANT** / cToken 数量保持不变
- **Exchange rate between cToken and underlying increases over time** / cToken 与底层资产的兑换率随时间上升
- Interest is realized at **redemption** / 利息在赎回时实现
- Example: 50,000 cUSDC initially redeem for 1,000 USDC → later redeem for 1,050 USDC

> **Key Trap / 关键陷阱**：Interest accrues through **rising exchange rate**, NOT by increasing token count / 利息通过兑换率上升累积，不是代币数量增加

### 7. Decentralized Derivatives = Oracle Dependency / 去中心化衍生品 = 预言机依赖

- Decentralized derivatives track underlying assets, events, or observable variables / 去中心化衍生品追踪底层资产、事件或变量
- They **inherently introduce oracle dependency** / 必然引入预言机依赖
- **Synthetix**: Minters bear exposure to the **entire system's debt pool**, not just their own position / 铸造者承担整个系统债务敞口，不只是自己的部分

### 8. Yield Farming — Sources of Yield / Yield Farming 收益来源

1. **Supply interest** — from lending protocols (Compound, Aave) / 供应利息，来自借贷协议
2. **Swap fee income** — from providing liquidity to AMM pools (Uniswap 0.3%) / 交易费，来自 AMM 池 LP
3. **Participation rewards** — from liquidity mining or governance token incentives / 参与奖励，来自流动性挖矿或治理代币

### 9. Four Security Risks / 四大安全风险

| Risk / 风险 | One-Liner / 一句话描述 |
|---|---|
| **Flash loan attack** / 闪电贷攻击 | Attacker borrows massive funds via flash loan → manipulates price or exploits protocol logic **in the same transaction** / 攻击者借大量资金，在同一交易中操纵价格或利用漏洞 |
| **Rug pull** / 卷款跑路 | Project administrators attract user funds then abandon project or drain assets / 项目方吸金后放弃项目或抽走资金 |
| **Reentrancy attack** / 重入攻击 | Called contract recursively calls back **before state is updated**, draining funds / 被调用合约在状态更新前递归回调，重复提款 |
| **Key exploit** / 密钥漏洞 | Private key, admin key or API key is compromised to modify contracts or drain funds / 私钥/管理员密钥被盗用来修改合约或抽走资金 |

### Most Likely Essay Questions / 最可能考的短答题（Ranked by Probability / 按概率排序）

**1. Compare DeFi and Traditional Finance / 比较 DeFi 和传统金融**
> DeFi uses smart contracts and decentralized networks to perform financial functions without relying on traditional intermediaries. Users bring their own wallets and identities, transactions are transparent and globally accessible. Traditional finance relies on centralized intermediaries, permissions and settlement infrastructure. DeFi can lower costs and increase innovation, but creates risks in governance, security, consumer protection and regulatory accountability.

**2. Name and explain three types of DEX / 说出并解释三种 DEX（Past Year Essay / 去年原题）**
> (1) **Order book DEX**: Makers place orders, takers accept, smart contract settles. On-chain orders are decentralized but costly; off-chain orders use relayers. (2) **AMM/CFMM**: Users trade against liquidity pools using invariants like `x*y=k`. Price from reserves, no oracle needed, arbitrage aligns with market. (3) **Reserve aggregation**: Liquidity providers advertise prices, smart contract selects best quote for atomic execution (e.g., Kyber). A fourth type is **P2P/OTC** (e.g., AirSwap).

**3. Explain AMM/CFMM and slippage with `x*y=k` / 用 x\*y=k 解释 AMM 和滑点**
> An AMM/CFMM is a smart contract liquidity pool where reserves satisfy `x*y=k`. Users trade against the pool — when one token is added, the other is output to maintain the invariant. The trade changes reserve ratios, so execution price differs from initial spot price. Larger trades relative to pool depth create larger non-linear price movement (slippage). Arbitrageurs align pool prices with market prices.

**4. Explain Terra's death spiral and propose a redesign / 解释 Terra 死亡螺旋并提出重设计**
> Terra used UST (stable unit) and LUNA (volatile absorber). When UST fell below $1, users burned UST to mint $1 worth of LUNA, creating LUNA sell pressure. During a large demand shock, massive UST exits expanded LUNA supply, crashed LUNA price, and required even more LUNA per redemption — a reflexive death spiral. Redesign: add circuit breakers, capped LUNA issuance, redemption queues, dynamic haircuts, and rule-based exogenous reserves. Trade-off: lower capital efficiency and less instant convertibility.

**5. Compare CDP and pooled lending, explain cTokens / 比较 CDP 和池化借贷，解释 cToken**
> A CDP (e.g., MakerDAO) lets users lock collateral and mint new debt tokens without a lender. Pooled lending (e.g., Compound) aggregates lenders' assets into pools — borrowers borrow existing assets, and rates vary with utilization. cTokens are minted when users supply to Compound; they represent proportional claims on the pool. cToken count stays constant while the exchange rate increases, so interest is realized at redemption.

**6. Explain the RWA tokenization process / 解释 RWA 代币化流程**
> RWA tokenization starts with legal and regulatory design, not technology. Steps: define asset and rights → regulatory classification → form legal vehicle (SPV/fund) → prepare offering documents → design token/smart contract → onboard compliant investors → manage post-issuance lifecycle → enable secondary trading/settlement/redemption. Key: tokenization improves access but does not eliminate credit, liquidity or legal risk.

**7. Discuss yield farming sources and risks / 讨论 yield farming 收益来源和风险**
> Yield sources: supply interest from lending (Compound/Aave), swap fees from AMM LPs, and participation rewards from liquidity mining. Risks: flash loan attacks (manipulate prices in single transaction), rug pulls (admins drain funds), reentrancy attacks (recursive calls before state update), key exploits, oracle manipulation, and unsustainable token rewards.

**8. Explain Singapore's position on CBDCs and stablecoins / 解释新加坡对 CBDC 和稳定币的立场**
> MAS sees stablecoins as combining fiat credibility with blockchain advantages, but requires regulation due to run-on-issuer risk. CBDC is a direct central bank liability. Wholesale CBDC can improve cross-border settlement. Retail CBDC is not urgent for Singapore because digital payments are already efficient, inclusion benefits are limited, and currency substitution risk is low. Project Orchid builds capabilities for future digital SGD use cases.
